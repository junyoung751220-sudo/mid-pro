# spec.md — 논술 원서 지원 도구 (nonsul-tracker)

## 개요

고3 학생이 논술 전형으로 지원할 서울 상위 15개 대학의 논술고사 일정, 학생부 반영 비율, 수능 최저학력기준을 한눈에 조회할 수 있는 웹 도구. 사용자는 (1) 전체 대학의 일정을 날짜순으로 훑어보거나, (2) 관심 학교·학과를 여러 개 입력해 상세 비교표를 보고, 날짜가 겹치는 지원을 즉시 인지할 수 있다. 조회 결과는 A4로 인쇄할 수 있다.

## 범위

### 포함
- 서울 소재 논술전형 상위 15개 대학(임시 목록, 실제 데이터는 추후 채움)을 대상으로 한 정적 데이터 조회
- "날짜순 보기": 전체 대학의 논술고사 일정을 날짜 순서로 나열한 리스트
- "학교·학과 검색 보기": 학교+학과 조합을 여러 개(최대 10개) 추가해 각각의 논술일, 수능최저, 논술:학생부 반영 비율을 표로 확인
- 검색 보기에서 추가한 항목들 간 논술일이 같은 날이면 해당 행을 빨간색으로 강조
- 학교는 목록에서, 학과는 드롭다운/자동완성으로 선택(자유 텍스트 입력 아님)
- 입력한 학과에 정확히 일치하는 데이터가 없을 때, 같은 계열(인문/상경/자연/공학 등 통상적 분류)의 대표 학과 데이터로 대체해서 보여주고 실제로 어떤 학과 데이터가 표시되는지 명시
- 대체할 계열 데이터도 없으면 "데이터 없음" 표시
- 날짜순 보기와 검색 보기 모두에 대해 A4 인쇄용 출력 버튼 제공
- 초기 데이터는 비어 있는 상태(학교 목록만 존재, 학과별 수치 데이터 없음)로 시작 — 화면에는 모든 조회 대상이 "데이터 없음"으로 표시됨

### 제외
- 진학사/메가스터디 자동 스크래핑 — 이용약관·로그인 이슈로 불안정하여 제외. 데이터는 사용자가 확인 후 수동으로 입력
- 로그인/계정/데이터 저장 동기화 — 단일 사용자가 로컬에서 쓰는 도구이므로 불필요
- 고사장 정보 — 시험 장소는 다루지 않음
- 수능 100% 반영, 학생부 100% 반영 등 논술 미포함 전형 — 이번 도구는 논술전형만 대상
- 겹침 판정에서 시간대(오전/오후) 구분 — 같은 날짜면 시간대와 무관하게 겹침으로 표시(단순화)

## 시나리오

### 1. 날짜순 보기로 전체 일정 확인
- **Given** 사용자가 도구를 열고 "날짜순 보기" 모드에 있음
- **When** 화면이 로드됨
- **Then** 15개 대학의 논술고사 일정이 날짜가 빠른 순서대로 한 줄씩 나열되고, 각 행에는 대학명과 논술일(날짜 + 오전/오후 + 정확한 시각)이 표시된다

성공 기준:
- [ ] 데이터가 있는 대학들은 날짜 오름차순으로 정렬되어 나열된다
- [ ] 아직 데이터가 없는 대학은 "데이터 없음"으로 표시되며 목록 하단(또는 별도 구획)에 나타난다

### 2. 학교·학과 검색으로 상세 정보 확인
- **Given** 사용자가 "검색 보기" 모드에서 학교 목록 중 하나를 고르고 그 학교의 학과 드롭다운에서 학과 하나를 선택함
- **When** "추가" 동작을 실행함
- **Then** 결과 표에 해당 학교·학과 행이 추가되고, 논술일, 수능최저(원문 텍스트), 논술:학생부 반영 퍼센트가 함께 표시된다

성공 기준:
- [ ] 학교="A대학교", 학과="경영학과" 선택 → 표에 "A대학교 / 경영학과" 행이 추가되고 논술일·수능최저·반영비율 값이 채워진다
- [ ] 해당 학과에 정확한 데이터가 없고 같은 계열 대표 데이터로 대체된 경우, 행에 실제 표시 중인 학과명(대체된 학과명)이 함께 표시된다
- [ ] 같은 계열 대표 데이터도 없는 경우 해당 필드들에 "데이터 없음"이 표시된다

### 3. 여러 학교 입력 시 논술일 겹침 강조
- **Given** 검색 보기 결과 표에 이미 논술일이 2026-09-20(오전, 09:00)인 "B대학교 / 국어국문학과" 행이 있음
- **When** 사용자가 논술일이 2026-09-20(오후, 14:00)인 "C대학교 / 사학과"를 추가함
- **Then** 두 행 모두 빨간색으로 강조되어 같은 날 논술이 겹친다는 것을 알 수 있다

성공 기준:
- [ ] 두 항목의 논술일(날짜만 비교, 시간대 무관)이 같으면 두 행 모두 빨간색 강조 스타일이 적용된다
- [ ] 겹치지 않는 다른 행에는 강조 스타일이 적용되지 않는다
- [ ] 겹치는 항목을 제거해 겹침이 해소되면 남은 행의 강조가 사라진다

### 4. 검색 결과 항목 개수 제한
- **Given** 검색 보기 결과 표에 이미 10개의 학교·학과 항목이 담겨 있음
- **When** 사용자가 11번째 항목을 추가하려고 시도함
- **Then** 추가가 차단되고 최대 개수(10개)에 도달했다는 안내가 표시된다

성공 기준:
- [ ] 10개가 담긴 상태에서 11번째 추가를 시도하면 표에 새 행이 생기지 않고 안내 문구가 나타난다
- [ ] 항목을 하나 제거해 9개가 되면 다시 추가할 수 있다

### 5. 결과를 A4로 인쇄
- **Given** 사용자가 검색 보기(또는 날짜순 보기)에서 결과를 확인 중임
- **When** "인쇄" 버튼을 클릭함
- **Then** 현재 보고 있는 화면(검색 결과 표 또는 날짜순 리스트)이 A4 한 페이지에 맞게 정리된 인쇄 미리보기/인쇄 다이얼로그가 열린다

성공 기준:
- [ ] 검색 보기에서 인쇄 클릭 → 현재 담긴 항목들의 표(학교·학과·논술일·수능최저·반영비율, 겹침 강조 포함)가 인쇄 레이아웃으로 나타난다
- [ ] 날짜순 보기에서 인쇄 클릭 → 날짜순 리스트가 인쇄 레이아웃으로 나타난다
- [ ] 인쇄 레이아웃에는 입력용 버튼·드롭다운 등 조작 UI가 나타나지 않는다

## 불변 규칙

- 논술일 표시는 항상 "날짜 + 오전/오후 + 정확한 시각(시:분)" 형식을 유지한다 — 화면 어디서 보여지든(날짜순 보기, 검색 보기, 인쇄) 동일한 표현을 쓴다
- 데이터가 없는 항목은 빈 칸이 아니라 항상 "데이터 없음"이라는 명시적 문구로 표시한다 — 사용자가 "아직 안 찾아본 것"과 "실제로 없는 것"을 구분하지 않아도 되게 한다
- 계열 대체 데이터가 표시될 때는 항상 실제로 표시 중인 학과명을 함께 노출한다 — 사용자가 원래 입력한 학과와 실제 표시된 데이터가 다르다는 것을 놓치지 않게 한다
